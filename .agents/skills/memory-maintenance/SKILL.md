# Memory Maintenance Skill

## Goal

Keep `MEMORY.md` durable, concise, current, and non-conversational.

## Rules

- Add only stable repo facts, repeated mistakes, durable constraints, or changed conventions.
- Do not add task notes, speculation, or chat summaries.
- Move transient findings to retrospectives, issues, or plans.
- Delete or replace deprecated guidance.

## Output

- Memory entries added
- Memory entries removed
- Reason each entry is durable
- Related validation or issue links
