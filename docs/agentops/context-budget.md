# Context Budget

Agents perform better when repository context is layered and selective.

## Rules

- `AGENTS.md` is a map, not a manual.
- Detailed guidance belongs in focused docs.
- Do not duplicate instructions across files.
- Split large documents by workflow or owner.
- Prefer registries, indexes, and checklists over long prose.
- Archive completed plans.
- Delete obsolete documents instead of keeping stale references.

## Warning signs

- Agents read many unrelated docs before simple tasks.
- Same rule appears in multiple places.
- `AGENTS.md` grows beyond a quick map.
- Old plans conflict with current docs.
- Skills and subagents duplicate each other.
